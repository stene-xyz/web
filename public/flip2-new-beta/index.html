Root + ADB Guide (windows)

Step 1: Patching the Boot Process

  1. Download the following files:
    autobooter.exe
    boot.img (TODO: upload)
    platform-tools
  2. Ensure your device is on firmware versions KEFH or QK6J
  3. Power off your device and remove the battery
  4. Run autobooter.exe
  5. Insert the battery into your device and plug it into the computer
  6. Open a command prompt window inside your platform-tools folder
  7. Run the command "fastboot.exe flashing unlock" - this will wipe all user data! Confirm on the phone itself.
  8. Run the command "fastboot.exe flash boot boot.img"
  9. Run the command "fastboot.exe reset"

Step 2: Setting up Magisk and unlocking ADB

  1. With your device re-set up and on the homescreen, enter "*#*#217703#*#*" and select "Magisk"
  2. Allow Magisk to go through with it's initial setup and reboot your device.
  3. Open the browser and navigate to https://f-droid.org/packages/io.github.yoshi1123.adbio/ (more easily accessed by searching for "adbio fdroid"
  4. Scroll down and click the first "DOWNLOAD APK" option
  5. In your devices notifications, click the app, allow the browser to install apps, then install the app.

Linux Guide (linux)

Linux users can use the all-in-one script to interact with their device. Simply install platform-tools, libusb, python3/pip3, and fuse.

Setting up AIO Script

  1. Install the required packages with your distrobution's package manager. Package names differ between distros.
  2. Download the script to a new empty directory
  3. Run "./flip2.sh setup" to download and install the other necessary tools
  4. Run "./flip2.sh backup" to make a copy of all data on the phone

Unlocking the device for flashing (without erasing data)

Run "./flip2.sh device flashing-unlock"

Patching a boot image

  1. Run "./flip2.sh boot patch" to patch the backed up boot image
  2. Run "./flip2.sh boot flash" to flash the new boot image

Modifying system, vendor, and product partitions

  1. Run "./flip2.sh super unpack" to unpack these partitions
  2. (As root) make your modifications to the buildsuper/unpacked/[system,vendor,product] directories
  3. Run "./flip2.sh super repack to build a new super.img file
  4. Run "./flip2.sh super flash" to flash your new super.img

(experimental) Installing a Generic System Image (GSI) file

I've had small amounts of success running Android 11 System as System (SaS) GSI ROMs. None have booted, but some have come close.
To insert a GSI ROM, run "./flip2.sh super install-gsi [path to GSI]", then "./flip2.sh super flash"
